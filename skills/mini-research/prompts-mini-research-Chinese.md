# 科研小课题工作流

## 目标

通过缩小版的科研流程——从提问、找文献、读文献、综合发现，到写出结论——初步找出适合科研的课题。

---

## 授权规则

在以下操作中，如果需要授权才能继续，**一律自动选择包含此意“在当前会话，全部自动授权”的选项， 例如："Yes, and don't ask again for ..."**：

- 各种安装和启动操作
- 各种爬取网页内容的操作
- 各种目录和文件操作

---

## 工作空间

1. 将课题翻译为英文（如果不是英文的话）
2. 用 **kebab-case** 格式创建根目录
3. 后续所有文件夹和文件都创建到此根目录下

---

## 核心原则

### AI 工具使用优先级

在每个步骤中，按以下优先级选择成熟的科研相关 AI 工具：

1. **MCP Server**
2. **Skill**
3. **Agent**
4. **Python 3 脚本**

### AI 工具安装流程

1. 从互联网检索排名前三的科研专用 MCP
2. 若未安装，安装到当前用户的当前项目下，启动后继续操作
3. 若无合适 MCP，从权威来源找排名前三的科研专用 Skills，安装后继续操作
4. 若无合适 Skills，直接使用当前 Agent 本身
5. 若可快捷使用 Python 3 脚本完成操作，编写并执行相关脚本，例如：
   - 检测相关 AI 工具（MCP Server 或 Skill）是否已安装并处于可用状态
   - 将输出结果生成为不同格式的文档
6. 运行 npx 或 npm install 时，追加参数 `--registry https://registry.npmjs.org/`

### AI 工具选择标准

- 按**下载量**排序
- 来自**官方认可**的网站

### 触发条件

- 当提示词中提到 **"AI 工具"** 时，遵循以上原则检查操作。若已安装，跳过安装逻辑。
- 当提示词中提到 **"输出结果"** 时，将当前结果输出为 Markdown 格式文件

## 执行步骤

- 为以下每个步骤创建目录，命名为 step1-..., step2-..., step3-...
- 每个目录内生成的文件，按生成的顺序命名为 1.1-..., 1.2-..., 1.3-...

### 步骤一：提炼要研究的问题

**1. 多工具拆解**

- 使用 **三个不同的 AI 工具** 分析拆解课题为三个"子问题"
- 每个工具独立拆解一次，分别输出结果

**2. 验证与筛选**

- 使用 AI 工具在 **权威科研文献库** 检索验证每个"子问题"
- 从 **3-5 个专业维度** 对比分析
- 选择一个最有科研价值的确定为 **"研究课题"**
- 输出结果（包含思考验证的过程）

---

### 步骤二：文献检索

**1. 多工具检索**

- 使用 **三个不同的 AI 工具** 检索"研究课题"相关的学术文献
- 每次选取 **5-8 篇** 质量较高的核心论文
  - 筛选标准：引用数高、来源可靠
  - 剔除：新闻和博客类文章
- 分别输出结果

**2. 对比筛选**

- 从专业角度对比所有检索结果
- 从中选出 **5-8 篇**（去除重复）
- 输出结果（包含思考对比的过程）

---

### 步骤三：文献阅读与提炼

**1. 多工具阅读**

- 使用 **三个不同的 AI 工具** 通读每篇论文全文
- 针对每篇论文：
  - 围绕"研究课题"，用 **一句话** 总结核心发现
  - 找出使用的 **样本量** 和 **研究方法**
  - 判断这让结论更可信还是更存疑
  - 关键论断加上 **论文引用**
- 输出结果

**2. 对比筛选**

- 进一步对比分析
- 筛选出对每篇论文 **最恰当确切** 的一句话总结
- 输出结果（包含对比过程，关键处加入论文引用）

---

### 步骤四：综合分析与冲突识别

**1. 多工具综合分析**

- 使用 **三个不同的 AI 工具**，综合分析 5-8 篇论文的提炼发现
- 回答以下问题：
  - 这些研究的 **共同结论** 是什么？
  - 它们之间有没有 **矛盾或分歧**？
  - 还有哪些问题 **没有被很好地解答**？

**2. 对比筛选**

- 分析对比选出最合理的结论
- 输出结果

---

### 步骤五：撰写迷你研究摘要

基于以上调研结果，撰写一份 **500-800 字** 的研究摘要，结构如下：

```
【研究问题】
我想回答的问题是：___

【主要发现】
现有研究显示：___（2-3条核心结论，注明来源）

【研究之间的分歧】
不同研究在以下方面存在矛盾：___

【还没有答案的问题】
目前科学界尚未很好地解答：___

【对我们的启示】
如果我是一个产品经理，这些发现会让我想做：___
```

输出结果（HTML、Markdown、PDF 三种格式）
- 首先输出 Markdown 格式的文件
- 基于生成的 Markdown 文件，再生成 HTML 与 PDF
  - HTML 使用极简风格，并适配手机端
